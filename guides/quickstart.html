Quickstart

Pick your platform. The same synthesizable kit drives all three — checkers and coverage come along on the Monitor.

1 · cocotb

Drive the transactor's signal-level FIFOs from Python. No SystemVerilog testbench required.

from fwvip_wb import WishboneInitiator, WishboneMonitor

async def test_rw(dut):
    init = WishboneInitiator(dut, clk=dut.clk, rst=dut.rst)
    mon  = WishboneMonitor(dut, clk=dut.clk)   # checkers + coverage
    await init.reset()
    await init.write(addr=0x40, data=0xCAFEF00D)
    assert await init.read(addr=0x40) == 0xCAFEF00D

2 · SystemVerilog / UVM

Instantiate the class-based agents; enable checks and coverage through the monitor config.

// build_phase
m_init = fwvip_wb_init_agent::type_id::create("m_init", this);
m_mon  = fwvip_wb_mon_agent ::type_id::create("m_mon",  this);

fwvip_wb_mon_cfg cfg = fwvip_wb_mon_cfg::type_id::create("cfg");
cfg.checks_enable   = 1'b1;
cfg.coverage_enable = 1'b1;   // collect protocol coverage
uvm_config_db#(fwvip_wb_mon_cfg)::set(this, "m_mon", "cfg", cfg);

// run_phase: drive sequences
fwvip_wb_write_seq::type_id::create("wr").start(m_init.sqr);
fwvip_wb_read_seq ::type_id::create("rd").start(m_init.sqr);

3 · Formal (SymbiYosys)

The transactor doubles as a protocol converter; the synthesizable checker becomes your formal property set.

# bind the synthesizable checker to the DUT bus
bind dut fwvip_wb_checker #(.ENABLED(1)) u_chk (.*);

# wishbone.sby
[options]
mode prove
[engines]
smtbmc
[script]
read -formal dut.v fwvip_wb_checker.v
prep -top dut

Same properties as simulation — authored once, run in both engines.
